## 📌 About

This repository contains my structured DSA solutions written in C++.

Every problem is solved in **multiple approaches** — starting from brute force,
moving to better solutions, and ending at the optimal.
Each file documents time/space complexity and the key learning from that problem.

The goal isn't to just pass test cases. It's to understand *why* the optimal works.

## 🧱 Solution Format

Every solution follows this structure:

```cpp
/*
==================================================
Problem: 217. Contains Duplicate
LeetCode: Easy

Approach 1: Brute Force     → O(n²) time, O(1) space
Approach 2: Sorting         → O(n log n) time, O(1) space
Approach 3: Hash Set        → O(n) time, O(n) space  ← Optimal

Learning:
- Brute force causes TLE on large inputs.
- Sorting makes duplicates adjacent — cleaner check.
- Hashing gives O(1) average lookup, best overall.
==================================================
*/
```
